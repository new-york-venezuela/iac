# Mailcow + Caddy on Hetzner Cloud — GitOps IaC

Fully automated, zero-secret-leak infrastructure for a self-hosted corporate email suite (Mailcow) and static website, deployed on Hetzner Cloud via Terraform + Ansible.

## Architecture

```
Internet
   │
   ▼  :80 / :443
┌─────────────────────────────────────────────┐
│  Caddy (native, host machine)               │
│  • Automatic TLS via Let's Encrypt (ACME)   │
│  • Serves static site from /var/www/...     │
│  • Reverse-proxies mail UI → localhost:8080 │
└──────────┬──────────────────────────────────┘
           │ 127.0.0.1:8080 (HTTP only)
           ▼
┌─────────────────────────────────────────────┐
│  Mailcow (Docker Compose, /opt/mailcow-*)   │
│  SKIP_LETS_ENCRYPT=y  HTTP_BIND=127.0.0.1  │
│  :25 :465 :587 :993 still public (SMTP/IMAPs)│
└──────────┬──────────────────────────────────┘
           │ Docker data-root
           ▼
┌─────────────────────────────────────────────┐
│  Hetzner Block Volume (50 GB ext4)          │
│  Mounted at /mnt/mailcow-data               │
│  Docker data-root = /mnt/mailcow-data/docker│
│  Backups at /mnt/mailcow-data/backups       │
└─────────────────────────────────────────────┘
```

## Tech Stack

| Layer | Technology |
|---|---|
| Cloud | Hetzner Cloud (CX22, fsn1) |
| IaC | Terraform ≥ 1.5 · hetznercloud/hcloud provider |
| Config Management | Ansible ≥ 9 · official [`mailcow.mailcow`](https://github.com/mailcow/mailcow-ansiblerole) role |
| Edge Proxy | Caddy 2 (automatic HTTPS, no certbot cron needed) |
| Mail Suite | Mailcow (mailcow-dockerized, Docker Compose v2) |
| Persistence | Hetzner Block Storage — Docker data-root on volume |
| Backups | `backup_and_restore.sh` cron → rclone offsite |
| Python Toolchain | uv + Python 3.13 (pinned via `.python-version` + `requires-python`) |

## Security Properties

- No secrets in git — all sensitive values via `.env` → `TF_VAR_*` environment variables
- `ansible/inventory.ini` is gitignored; Terraform regenerates it on every `apply`
- SSH firewall rules are parameterised (`allowed_ssh_cidrs`) — lock to office IPs in production
- Caddy handles TLS automatically; no manual cert rotation
- Docker log rotation enforced via `daemon.json` (50 MB / 3 files)
- Automatic OS security updates via `unattended-upgrades`
- `fail2ban` installed for brute-force protection
- 4 GB swapfile prevents OOM kills during mail scanning spikes

## Replicating for Another Company

1. Copy `.env.example` → `.env`, fill in new company values
2. Change `TF_VAR_company_name`, `TF_VAR_domain`, `TF_VAR_admin_email`
3. Upload an SSH key to Hetzner and set `TF_VAR_ssh_key_name`
4. Run `terraform apply`, then `ansible-playbook` — see INSTRUCTIONS.md
